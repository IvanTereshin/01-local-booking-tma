# План разработки: Local Booking TMA

## 1. Идея

Мини-приложение для быстрой записи к специалисту внутри Telegram. Пользователь открывает TMA из бота, выбирает услугу, специалиста, дату и время. После записи бот присылает подтверждение и напоминание.

## 2. MVP

- Экран выбора услуги.
- Экран выбора специалиста.
- Календарь со свободными датами.
- Список свободных временных слотов.
- Форма подтверждения записи.
- Экран "Мои записи".
- Админ-экран с расписанием и заявками.
- Уведомления через Telegram-бота.

## 3. Роли

- Клиент: выбирает услугу и записывается.
- Специалист: видит свои записи.
- Администратор: управляет услугами, специалистами, расписанием и статусами записей.

## 4. Основные экраны

1. Главная: ближайшие свободные окна, популярные услуги.
2. Услуги: список, цена, длительность, описание.
3. Специалисты: фото, рейтинг, специализация, ближайший слот.
4. Календарь: доступные даты и слоты.
5. Подтверждение: услуга, специалист, дата, цена, комментарий.
6. Мои записи: активные, завершенные, отмененные.
7. Админка: заявки на день, управление слотами, блокировка времени.

## 5. Данные

- `users`: telegram_id, name, username, role.
- `services`: title, description, duration_minutes, price.
- `specialists`: name, photo_url, bio, service_ids.
- `availability`: specialist_id, date, start_time, end_time, is_blocked.
- `bookings`: user_id, specialist_id, service_id, starts_at, status, comment.
- `notifications`: booking_id, type, scheduled_at, sent_at.

## 6. Backend API

- `GET /services`
- `GET /specialists?serviceId=...`
- `GET /availability?specialistId=...&date=...`
- `POST /bookings`
- `GET /me/bookings`
- `PATCH /bookings/:id/cancel`
- `GET /admin/bookings`
- `PATCH /admin/availability`

## 7. Telegram-интеграция

- Bot menu button: "Записаться".
- Проверка `initData` на backend.
- Сообщение после создания записи.
- Напоминание за 24 часа и за 2 часа.
- Inline-кнопки в сообщении: "Перенести", "Отменить", "Открыть запись".

## 8. Технологии

- Frontend: React, Vite, TypeScript.
- UI: Telegram UI или свой lightweight design system.
- Backend: Node.js + Fastify/NestJS.
- DB: PostgreSQL.
- Bot: grammY или Telegraf.
- Deploy: Docker, HTTPS-домен, webhook.

## 10. Критерии готовности

- Пользователь может создать запись без внешнего браузера.
- Админ может увидеть и изменить запись.
- Бот отправляет подтверждение.
- Нельзя записаться на уже занятый слот.
- UI нормально выглядит в светлой и темной теме Telegram.
