# Local Booking TMA

Production-ready demo Telegram Mini App для записи к специалисту. Первый экран — рабочее приложение, не лендинг: пользователь сразу выбирает услугу, специалиста, дату, слот и подтверждает запись.

## Что внутри

- Mobile-first app shell под Telegram Mini Apps: safe area, компактная шапка, нижняя навигация.
- Клиентский booking-flow: услуга -> специалист -> дата -> слот -> подтверждение -> success-state.
- Экран "Мои записи": активные и прошлые записи, отмена активной записи.
- Админ-экран: выбор дня, список записей, смена статусов, блокировка и разблокировка слота.
- Проверка занятости слота: нельзя забронировать или заблокировать уже занятое время.
- Моковый API с небольшой задержкой, чтобы интерфейс вел себя как живой продукт.
- Сохранение записей и блокировок в `localStorage`.
- Fallback для отсутствующих изображений без битых картинок.

## Демо-flow

1. Откройте вкладку `Запись`.
2. Выберите услугу, например `Стрижка и укладка`.
3. Выберите специалиста.
4. Выберите дату и свободный слот.
5. Нажмите `Подтвердить запись`.
6. Перейдите в `Мои`, чтобы увидеть созданную запись.
7. В `Админ` можно сменить статус записи или заблокировать свободный слот.

## Данные и состояние

Демо-данные лежат в `src/data.ts`: услуги, специалисты, расписание, стартовые записи и заблокированные слоты.

Состояние хранится в `localStorage`:

- `local-booking-tma:bookings`
- `local-booking-tma:blocked-slots`

Чтобы сбросить демо, очистите эти ключи в DevTools или очистите localStorage для localhost.

## Production notes

- Сейчас backend заменен моковым слоем `src/mockApi.ts`. Для продакшена его можно заменить на REST/RPC API без переписывания UI.
- Проверка занятости слота должна быть продублирована на сервере, потому что клиентская проверка нужна только для UX.
- Для реального продукта нужны роли пользователя: клиент, специалист, администратор.
- Для платежей, предоплаты и штрафов за отмену нужно отдельное состояние заказа.
- Для расписания лучше хранить рабочие часы, исключения, отпуска и буферы между записями на backend.
- Для уведомлений нужен Telegram bot backend: подтверждение, перенос, отмена, напоминания.

## Telegram integration notes

- `src/telegram.ts` вызывает `Telegram.WebApp.ready()` и `Telegram.WebApp.expand()`, если Mini App открыт внутри Telegram.
- Theme params пробрасываются в CSS-переменные: фон, текст, muted-цвет и цвет кнопки.
- Для продакшена нужно валидировать `initData` на backend.
- URL приложения нужно добавить в BotFather как Web App / Menu Button.
- В Telegram стоит проверять светлую и темную тему, safe-area на iOS и поведение клавиатуры на форме комментария.
