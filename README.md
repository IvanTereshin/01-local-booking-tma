# Atelier Vera — Local Booking TMA

Премиальный frontend-концепт Telegram Mini App для записи в салон, клинику или частную практику. Продукт превращает выбор услуги, специалиста и времени в один спокойный сценарий, а результат оформляет как узнаваемый booking pass.

**Публичное демо:** [tma-01-booking.ivantereshin-test.store](https://tma-01-booking.ivantereshin-test.store/)

![Atelier Vera — desktop-презентация и Telegram Mini App](docs/assets/atelier-vera-overview.jpg)

## Задача продукта

Обычная онлайн-запись часто выглядит как техническая форма: длинный список услуг, непонятные статусы слотов и слабое подтверждение результата. Atelier Vera показывает другой подход — интерфейс одновременно продаёт атмосферу бренда и помогает без лишних решений закончить запись.

Концепция подходит для:

- beauty-студий и барбершопов;
- косметологических и медицинских клиник;
- массажных и wellness-пространств;
- частных специалистов с собственным расписанием.

Визуальный слой можно адаптировать под конкретный бренд, а демонстрационный источник данных — заменить реальным API без перестройки пользовательского сценария.

## Основной сценарий

1. Клиент знакомится с пространством и выбирает услугу.
2. Сравнивает специалистов по профилю, рейтингу и специализации.
3. Выбирает дату и видит честные состояния времени: свободно, занято, лист ожидания или конфликт.
4. Проверяет детали, выбирает демонстрационный способ оплаты и подтверждает запись.
5. Получает booking pass и может управлять записью во вкладке «Мои».

## Что реализовано

### Клиентская часть

- пятиэтапная шкала: услуга → мастер → дата → время → готово;
- editorial-прайс без визуально тяжёлых карточек;
- крупные профили специалистов;
- доступные состояния слотов с цветом, текстом и `aria`-атрибутами;
- демонстрация оплаты через Telegram Stars, СБП и ЮKassa;
- подтверждение в виде перфорированного booking pass;
- активные и прошлые записи, перенос и отмена;
- лист ожидания и защита от повторного выбора занятого времени.

### Операционная часть

- календарь администратора;
- список записей выбранного дня;
- изменение статуса визита;
- блокировка и разблокировка слотов;
- сохранение записей и ограничений в `localStorage`;
- моковый API с небольшой задержкой для правдоподобного поведения интерфейса.

### Интерфейс

- mobile-first компоновка с safe area для Telegram;
- презентационный desktop-frame для продаж и портфолио;
- палитра warm ivory / espresso / dusty rose и один copper-акцент;
- адаптивная типографика и интерактивные зоны не меньше мобильного стандарта;
- fallback-состояния для изображений;
- поддержка `prefers-reduced-motion`.

## Архитектура демо

```text
React UI
  ├── booking-flow и навигация
  ├── mockApi.ts — имитация сетевых операций
  ├── data.ts — услуги, мастера и расписание
  ├── localStorage — записи и заблокированные слоты
  └── telegram.ts — адаптация к Telegram WebApp
```

Граница проекта намеренная: это frontend-demo, который проверяет дизайн, взаимодействия и ценность сценария. В репозитории нет production-backend, реального эквайринга и доверенной серверной проверки `Telegram.initData`.

При подключении backend UI может остаться прежним. Заменить нужно слой `src/mockApi.ts`, а сервер должен стать единственным источником истины для расписания, блокировок, платежей и ролей.

## Локальный запуск

Требования: актуальная LTS-версия Node.js и npm.

```bash
npm ci
npm run dev
```

Vite запустит приложение на локальном адресе, указанном в терминале.

Полная проверка перед публикацией:

```bash
npm run check
```

Команда последовательно запускает ESLint, regression-тесты пользовательского сценария, TypeScript и production build.

## Как проверить демо

1. Откройте вкладку «Запись».
2. Выберите «Консультация по окрашиванию» и специалиста.
3. Выберите дату и свободный слот.
4. В блоке оплаты выберите ЮKassa и отправьте mock-код подтверждения.
5. Проверьте booking pass во вкладке «Мои».
6. Откройте «Админ», смените статус записи или заблокируйте свободное время.

- `local-booking-tma:blocked-slots`.

Для чистого прохождения удалите эти ключи в DevTools или очистите данные сайта.

## Telegram Mini App

`src/telegram.ts` вызывает `Telegram.WebApp.ready()` и `Telegram.WebApp.expand()`, когда приложение открыто внутри Telegram, и переносит theme params в CSS-переменные.

Для production-подключения потребуется:

- зарегистрировать Web App / Menu Button в BotFather;
- валидировать `initData` на backend;
- проверить safe area, клавиатуру и обе темы на реальных iOS/Android-устройствах;
- отправлять подтверждение, перенос и напоминания через Telegram-бота.

## Roadmap до production

1. **Backend-контракт:** пользователи, роли, филиалы, специалисты, услуги и расписание.
2. **Надёжное бронирование:** серверная транзакция, временный hold слота и защита от двойной записи.
3. **Платежи:** реальный order lifecycle, webhook эквайринга, возвраты и правила отмены.
4. **Расписание:** рабочие часы, исключения, отпуска и буферы между услугами.
5. **Коммуникации:** подтверждения, переносы и напоминания через Telegram.
6. **Эксплуатация:** аудит действий администратора, мониторинг, аналитика и резервное копирование.
